## `// WHO AM I`

> 문제는 늘 내 경험에서 시작합니다. 그리고 라이브러리를 가져다 쓰는 데서 멈추지 않고
> **원리(R1CS, QAP, bilinear pairing 같은 것까지)** 를 직접 파고듭니다.
> 동시에 **무엇을 만들지뿐 아니라 무엇을 멈출지** 도 판단하는 개발자가 되려 합니다.

- 🔐 **데이터 주권** — 얼굴도, 파일도 서버가 아닌 *내 기기 안에서* 처리되어야 한다고 믿습니다
- 🧠 원리까지 파고드는 학습 · 작동하는 시스템으로 구현
- ⚖️ 법적·윤리적 경계를 인지하고 멈출 줄 아는 판단

## `// FEATURED PROJECTS`

### 🛡️ PrivyFace — *얼굴정보를 서버에 보내지 않는 신원 인증*
> 영지식 증명(ZK-SNARK)으로 "내가 DB에 등록된 사람임"을 얼굴 데이터 노출 없이 증명.
> MediaPipe로 얼굴 특징 추출 → o1js로 ZK Proof 생성 → 서버는 결과만 검증.

`ZK-SNARK` `o1js` `MediaPipe` `Merkle Tree` `React`
[`→ 레포 보기`](https://github.com/kingcat47/privyface)

### 🚒 Oliver — *화재 진압 자율주행 로봇 관제 시스템* 🏆
> KAIST 총장상 수상작. 119 실시간 관제 대시보드(지도 기반 UI · 열화상 · WebSocket) 프론트엔드 전담.

`React` `WebSocket` `실시간 대시보드`
[`→ 레포 보기`](https://github.com/oliver-fire/oliver)

### 🖥️ CPU Simulator — *C로 만든 모의 CPU + 실시간 시각화*
> 명령어셋 설계부터 fetch–decode–execute 사이클까지. 레지스터·메모리 흐름을 React로 시각화한 교육용 시뮬레이터.

`C` `React` `컴퓨터 구조`
[`→ 레포 보기`](https://github.com/kingcat47/cpu)

### 🗂️ CHISA — *온디바이스 LLM으로 파일을 정리하는 에이전트*
> 로컬 LLM + RAG로 기기 안 파일을 분석해 일관된 이름·계층으로 자동 정리. 데이터는 서버로 나가지 않음.

`Kotlin` `Local LLM` `RAG`
[`→ 레포 보기`](https://github.com/kingcat47/CHISA)
